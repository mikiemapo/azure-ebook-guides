üö® CRITICAL: RTO vs RPO - Storage Replication Strategy

Your Weak Spot: Storage domain accuracy at 43% - This is your #1 priority!

Key Concepts:

  • RTO (Recovery Time Objective): How long until systems are back online
  • RPO (Recovery Point Objective): How much data loss is acceptable
  • Storage Replication: LRS ‚Üí ZRS ‚Üí GRS ‚Üí RA-GRS (increasing DR capability)
üéØ Exam Focus: You WILL get scenario questions asking you to choose between GRS vs RA-GRS based on RTO requirements. RA-GRS = Read access during outage (lower RTO).

üè™ Azure Storage Replication for DR

üè† LRS (Locally Redundant Storage)

Copies: 3 within same datacenter

Protection: Hardware failures only

RTO: Minutes (if datacenter survives)

RPO: Minimal

Use Case: Non-critical data, cost-sensitive

üè¢ ZRS (Zone-Redundant Storage)

Copies: 3 across availability zones

Protection: Datacenter failures

RTO: Minutes to hours

RPO: Minimal

Use Case: High availability within region

üåç GRS (Geo-Redundant Storage)

Copies: 3 local + 3 in paired region

Protection: Regional disasters

RTO: Hours (manual failover)

RPO: 15 minutes

Use Case: Critical data, regional protection

üìñ RA-GRS (Read-Access GRS)

Copies: Same as GRS + read access

Protection: Regional disasters

RTO: Minutes for read, hours for write

RPO: 15 minutes

Use Case: Apps needing read during outage

üéØ Exam Scenario: "Company needs to continue serving customers during primary region outage" = RA-GRS (read access during outage)

Azure CLI - VM Backup Configuration:
# Create Recovery Services Vault
az backup vault create --name myVault --resource-group myRG --location eastus

# Enable backup for VM
az backup protection enable-for-vm --vault-name myVault --vm myVM --policy-name DefaultPolicy

# Trigger backup
az backup protection backup-now --vault-name myVault --container-name myVM --item-name myVM

üîÑ Azure Site Recovery (ASR)

Orchestrates replication, failover, and recovery of VMs and applications.

Scenario Source Target RTO RPO
Azure to Azure Azure VMs Secondary Azure region 15-30 minutes 5 minutes
VMware to Azure On-premises VMware Azure 30-60 minutes 15 minutes

üéØ ASR Exam Points:
  • ASR provides automated failover - much faster than manual backup restore
  • Supports failback to original location after issue resolved
  • Includes recovery plans for multi-tier applications
  • Can integrate with Traffic Manager for DNS failover

üìã DR Testing and Validation

üéØ Test Types (Exam Favorite!):
  • Test Failover: Non-disruptive test of ASR setup
  • Planned Failover: Graceful shutdown and switch
  • Unplanned Failover: Emergency failover during actual disaster
  • Failback: Return to original location after recovery

DR Testing Best Practices

  • Regular Testing: Quarterly DR drills minimum
  • Document Results: RTO/RPO actual vs target
  • Update Procedures: Based on test findings
  • Stakeholder Training: Ensure team knows procedures
