Flashcard: Event Grid Subscription to Blob Storage

Lab Objective & Workflow

Connect a Blob Storage account to an Event Grid topic and route events to an Azure Web App running the Event Grid Viewer. Validate end-to-end flow by uploading blobs.

  1. Create Storage Account and Container.
  2. Deploy Event Grid Viewer Web App.
  3. Create Event Grid webhook subscription pointing to the web app.
  4. Upload blob and verify event in the viewer.

Storage Account & Container Creation 


$resourceGroup = "rg_centralus_xxx"
$location = "centralus"
$storageName = "mikestorage123"

New-AzStorageAccount -ResourceGroupName $resourceGroup `
  -Name $storageName -Location $location `
  -SkuName Standard_LRS -Kind BlobStorage `
  -AccessTier Hot -AllowBlobPublicAccess $false

$ctx = (Get-AzStorageAccount -ResourceGroupName $resourceGroup -AccountName $storageName).Context
New-AzStorageContainer -Name "gridcontainer" -Context $ctx

Deploy Event Grid Viewer

Used the ARM template from GitHub. Important: Web App names must be globally unique and only contain lowercase letters, numbers, and hyphens. 


$sitename = "mikesite123"

New-AzResourceGroupDeployment `
  -ResourceGroupName $resourceGroup `
  -TemplateUri "https://raw.githubusercontent.com/Azure-Samples/azure-event-grid-viewer/master/azuredeploy.json" `
  -siteName $sitename `
  -hostingPlanName viewerhost `
  -branch "main" -sku "B1"

Create Event Grid Subscription


$storageId = (Get-AzStorageAccount -ResourceGroupName $resourceGroup -AccountName $storageName).Id
$endpoint = "https://$sitename.azurewebsites.net/api/updates"

$obj = New-AzEventGridWebHookEventSubscriptionDestinationObject -EndpointUrl $endpoint

New-AzEventGridSubscription `
  -EventSubscriptionName "gridBlobQuickStart" `
  -Destination $obj `
  -Scope $storageId
