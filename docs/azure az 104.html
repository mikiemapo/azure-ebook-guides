Study Guide for Microsoft Azure Administrator (AZ-104)

Comprehensive preparation guide for the AZ-104 certification exam

Key Terms

  • Azure Backup - A service that provides data protection and recovery capabilities for Azure resources

    Critical for maintaining business continuity and disaster recovery in Azure environments

  • Azure Bastion - A fully managed service that provides secure and seamless RDP and SSH access to virtual machines

    Important for securing access to virtual machines without exposing them directly to the internet

  • Azure Container Apps - A fully managed serverless container service for building and deploying modern applications

    Enables deployment and management of containerized applications without managing infrastructure

  • Azure Container Instances - A service that allows running containers in Azure without managing servers or container orchestration

    Provides quick and simple container deployment without infrastructure management overhead

  • Azure DNS - A hosting service for DNS domains that provides name resolution using Microsoft Azure infrastructure

    Essential for managing domain names and DNS records in Azure environments

  • Azure Monitor - A comprehensive solution for collecting, analyzing, and acting on telemetry from cloud and on-premises environments

    Critical for monitoring performance and health of Azure resources

  • Azure Policy - A service used to create, assign, and manage policies that control or audit resources

    Helps enforce organizational standards and assess compliance across Azure resources

  • Azure Resource Manager (ARM) templates - JSON files that define infrastructure and configuration for Azure deployments

    Enables infrastructure as code and automated deployment of Azure resources

  • Azure Site Recovery - A disaster recovery service that ensures business continuity by keeping applications running during outages

    Critical for maintaining business operations during regional failures or disasters

  • Azure Storage Explorer - A standalone application for managing Azure Storage data across multiple accounts

    Provides a graphical interface for managing various types of Azure storage resources

  • Bicep - A domain-specific language for deploying Azure resources, which simplifies ARM template authoring

    Provides a more developer-friendly approach to infrastructure as code in Azure

  • Network Security Groups (NSGs) - Virtual firewall rules that control inbound and outbound network traffic to Azure resources

    Fundamental to implementing network security in Azure environments

  • Recovery Services vault - A storage entity in Azure that houses backup data and recovery points

    Central component for implementing backup and disaster recovery solutions

  • Shared Access Signature (SAS) - A URI that grants restricted access rights to Azure Storage resources

    Enables secure, delegated access to storage resources without sharing account keys

  • Virtual Machine Scale Sets - A group of identical, load-balanced VMs that can automatically increase or decrease in response to demand

    Enables automatic scaling of applications and improved availability

Key concepts

  • Azure Administrator Exam (AZ-104) - A certification exam that tests expertise in implementing, managing, and monitoring an organization's Microsoft Azure environment

    Validates skills required for Azure administration roles and demonstrates proficiency in core Azure services

  • Identity and Governance (20-25%) - Management of Azure identities, roles, subscriptions, and governance policies

    Critical for controlling access, managing resources, and maintaining security compliance in Azure environments

  • Storage Implementation (15-20%) - Configuration and management of Azure storage accounts, files, and blob storage

    Essential for managing data storage solutions and ensuring proper access controls in Azure

  • Compute Resources (20-25%) - Deployment and management of virtual machines, containers, and app services

    Core component for running applications and workloads in Azure

  • Virtual Networking (15-20%) - Configuration and management of virtual networks, security, and connectivity

    Fundamental for establishing secure communication between Azure resources

  • Monitoring and Maintenance (10-15%) - Implementation of monitoring solutions, backup, and disaster recovery

    Ensures reliability and availability of Azure resources

  • Azure Resource Manager (ARM) Templates - JSON files used for infrastructure as code deployment in Azure

    Enables automated and consistent resource deployment across Azure environments

  • Microsoft Entra ID - Azure's cloud-based identity and access management service

    Primary system for managing user access and authentication in Azure

  • Azure Monitor - Platform for collecting, analyzing, and responding to monitoring data

    Critical for maintaining performance and health of Azure resources

  • Azure Backup and Site Recovery - Services for data backup and disaster recovery solutions

    Essential for business continuity and data protection strategies

Azure Identity and Governance

Managing users, access controls, and governance policies in Azure

This section covers the fundamental aspects of identity management and governance in Azure. You'll learn how to manage Microsoft Entra users and groups, including creating and managing user accounts, assigning licenses, and configuring self-service password reset. The section also details Azure role-based access control (RBAC), implementing Azure Policy, and managing subscriptions and costs. Understanding these concepts is crucial for maintaining security and compliance in Azure environments. You'll explore how to use management groups to organize resources effectively and implement cost control measures through budgets and alerts.

Key Takeaways:

  • Master Microsoft Entra ID user and group management including license assignment
  • Understand and implement role-based access control across different scopes
  • Learn to use Azure Policy and resource tags for governance
  • Implement cost management through budgets and Azure Advisor
  • Configure and manage management groups for resource organization

Azure Storage Solutions

Implementing and managing Azure storage services

This section focuses on Azure storage solutions, covering configuration and management of storage accounts, access control, and different storage services. You'll learn how to implement security measures like storage firewalls and SAS tokens, manage access keys, and configure various storage redundancy options. The section covers both Azure Files and Blob Storage, including advanced features like lifecycle management, versioning, and soft delete. Understanding storage tiers and data protection mechanisms is essential for optimal cost management and data security.

Key Takeaways:

  • Configure and secure Azure Storage accounts with appropriate access controls
  • Implement shared access signatures and storage policies
  • Master Azure Files and Blob Storage configuration and management
  • Understand storage redundancy options and their use cases
  • Implement lifecycle management and versioning for blob storage

Azure Compute Resources

Deploying and managing various compute services in Azure

This section covers the deployment and management of Azure compute resources, including virtual machines, containers, and App Services. You'll learn how to use ARM templates and Bicep files for infrastructure as code, manage virtual machines and their components, and work with container services like Azure Container Instances and Container Apps. The section also details App Service configuration, including scaling, networking, and deployment slots. Understanding these compute options and their management is crucial for building and maintaining Azure infrastructure.

Key Takeaways:

  • Master ARM templates and Bicep files for resource deployment
  • Configure and manage virtual machines and scale sets
  • Implement container solutions using Azure Container Instances and Apps
  • Configure and manage Azure App Services effectively
  • Understand different compute options and their appropriate use cases

Virtual Networking

Implementing and managing Azure virtual networks

This section focuses on Azure networking concepts and implementation. You'll learn how to configure and manage virtual networks, implement network security through NSGs and application security groups, and set up secure access using Azure Bastion. The section covers DNS configuration, load balancing, and troubleshooting network connectivity issues. Understanding these networking concepts is essential for creating secure and efficient Azure infrastructure that meets organizational requirements.

Key Takeaways:

  • Configure and manage virtual networks and peering
  • Implement network security using NSGs and application security groups
  • Set up secure access with Azure Bastion and private endpoints
  • Configure Azure DNS and load balancing solutions
  • Master network troubleshooting and monitoring

Monitoring and Recovery

Implementing monitoring, backup, and disaster recovery solutions

This section covers Azure monitoring and recovery solutions, including Azure Monitor, backup services, and disaster recovery options. You'll learn how to configure and interpret monitoring metrics, set up alerts and action groups, and implement backup solutions using Recovery Services vaults. The section also covers Azure Site Recovery for disaster recovery scenarios and failover procedures. Understanding these monitoring and recovery concepts is crucial for maintaining highly available and resilient Azure infrastructure.

Key Takeaways:

  • Configure and use Azure Monitor for comprehensive monitoring
  • Implement backup solutions using Azure Backup
  • Configure disaster recovery using Azure Site Recovery
  • Set up effective alerting and monitoring strategies
  • Master backup and recovery procedures for Azure resources
