Enhanced Security for Gen2 VMs

A security capability for Azure Generation 2 VMs designed to protect against advanced and persistent attack vectors like bootkits and rootkits by ensuring a verified boot path from firmware to OS.

Trusted LaunchVerifies Boot Chain
Securely Booted Gen2 VM
  • Gen2 VMs Only: Exclusively for Generation 2 Azure Virtual Machines and VM Scale Sets.
  • Defense Against Boot Malware: Protects boot loaders, OS kernel, and early drivers from tampering before the OS fully loads.
  • Protects Secrets: Enables secure storage and use of keys, certificates, and secrets within the VM using vTPM capabilities.
  • Becoming Standard: Increasingly the default security setting for new Gen2 VMs.

Implication: Provides foundational compute layer security, significantly enhancing defense-in-depth from the moment the VM powers on.

Secure Boot + vTPM + Attestation

A combination of features working together to verify boot integrity. They can often be enabled separately but provide the strongest protection when combined.

Verifying Software Signatures

A platform firmware feature (virtual UEFI) that ensures only trusted, digitally signed software (bootloaders, OS kernel, drivers) can run during the VM boot process.

  • Gatekeeper Role: Prevents unsigned or maliciously signed code from executing early in the boot sequence.
  • Defense against Rootkits/Bootkits: Primary protection against malware that tries to load before the OS and hide.
  • Foundation for Attestation: Must be enabled for successful boot integrity attestation.

Implication: Hardens the VM by ensuring only authorized code initiates the boot process.

Secure Vault & Boot Measurement

A virtualized Trusted Platform Module (TPM 2.0 spec) providing a secure cryptoprocessor in the VM's virtual hardware. Acts as a secure vault.

  • Secure Key & Secret Storage: Used for BitLocker keys, platform health checks, and secure storage of other secrets.
  • Boot Integrity Measurement: Takes cryptographic "fingerprints" (hashes) of the entire boot chain: firmware, bootloader, OS kernel, critical drivers.
  • Enables Attestation: These secure measurements form the basis for verifying boot integrity.

Implication: Provides hardware-backed security for sensitive data and enables verifiable boot integrity.

Verifying Boot Integrity Remotely

The process of remotely verifying that the VM booted securely, based on the vTPM's measurements. Integrates with Microsoft Defender for Cloud.

  • Guest Attestation Extension: An optional VM extension. Microsoft Defender for Cloud recommends installing it. It allows Azure services to securely query the vTPM for boot measurements.
  • Microsoft Defender for Cloud Integration:
    • Uses vTPM measurements for guest attestation.
    • Performs threat identification: if measurements don't match expected baselines, it suggests tampering/malware and can raise an alert.
    • Provides security posture management and recommendations.

Implication: Provides ongoing assurance and monitoring of VM boot integrity, integrating into broader cloud security posture management.

Configuration Steps

  • New Gen2 VMs:
    • In Azure portal, during VM creation, select "Security type" as Trusted launch virtual machines.
    • Check boxes to "Enable Secure Boot" and "Enable vTPM".
    • Optionally enable "Guest Attestation" (recommended by Defender for Cloud).
  • Existing Gen2 VMs:
    • VM must be stopped (deallocated).
    • VM size must support Trusted Launch. If not, resize to a compatible Gen2 size first.
    • Enable via VM's configuration settings (Security or Advanced section in portal).
    • Warning: Do not resize a Trusted Launch VM to an unsupported size later, as it can break the configuration.

AZ-104 Focus: Knowing it's for Gen2 VMs, the portal options to enable Secure Boot & vTPM, and the critical prerequisites for existing VMs (stopped, compatible size).

Core Understanding for the Exam

  • Trusted Launch enhances security for Generation 2 VMs only.
  • Protects against boot-level malware (bootkits, rootkits) by verifying the entire boot chain before the OS loads.
  • Secure Boot: Ensures only digitally signed and trusted software (firmware, OS loader, kernel, drivers) runs at boot.
  • vTPM: Securely measures boot components (creates cryptographic fingerprints) and can store sensitive data like encryption keys.
  • Attestation: Process of verifying boot integrity using vTPM measurements, often facilitated by the Guest Attestation Extension and integrated with Microsoft Defender for Cloud for monitoring and alerts.
  • Know how to enable for new and existing (stopped, compatible Gen2 size) VMs.
  • It's a foundational security layer, part of Azure's defense-in-depth model for compute.
Trusted Launch secures the VM's boot process. It should be complemented by other security measures (NSGs, Azure Firewall, endpoint security software, regular patching, strong identity & access management) for comprehensive protection once the OS is running.
