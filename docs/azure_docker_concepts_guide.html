Understanding Docker & Its Role in Azure

A conceptual overview of Docker, its key components, and how it's used with Azure services like ACI, AKS, and App Service. Essential for AZ-104. Click/Tap headers.

An Open Platform for Containers

Docker is an open-source platform that enables developers to easily develop, ship, and run applications inside isolated environments called containers.

It packages software into standardized units that combine the application source code with all the operating system (OS) libraries and dependencies required to run the code in any environment.

Instructions to Build an Image

A Dockerfile is a simple text file that contains a list of commands that Docker uses to assemble a container image automatically.

  • Defines the base OS image (e.g., Ubuntu, Alpine).
  • Specifies dependencies to install (e.g., Python, Node.js, libraries).
  • Copies application code into the image.
  • Sets the command to run when the container starts.
  • Example commands: FROM, RUN, COPY, WORKDIR, CMD, ENTRYPOINT.

AZ-104 Context: You won't write complex Dockerfiles, but understand its role as the source for building images.

Immutable Application Package

A Docker image is a lightweight, standalone, executable package that includes everything needed to run a piece of software: the code, a runtime, libraries, environment variables, and config files.

  • Layered Filesystem: Images are built in layers, making them efficient for storage and distribution. Only changed layers are updated/transferred.
  • Immutable: Once an image is built, it doesn't change. If you need to make changes, you build a new image (often with a new version tag).
  • Tagged: Images are typically tagged with versions (e.g., myapp:latest, myapp:1.0).

A Running Instance of an Image

A container is a live, running instance of a Docker image. You can have many containers running from the same image.

  • Isolation: Containers run in isolated environments on the host OS, sharing the host's kernel but having their own process space, filesystem, and network interface.
  • Ephemeral by Default: Data written inside a container's writable layer is lost when the container stops, unless persistent storage (volumes) is configured.
  • Can be started, stopped, moved, and deleted quickly.

Storing and Distributing Images

A container registry is a storage system for Docker images. You push images to a registry and pull images from a registry.

  • Public Registries: E.g., Docker Hub (widely used for open-source and public images).
  • Private Registries: For storing proprietary or internal application images.
    • Azure Container Registry (ACR): Microsoft's managed private Docker registry service in Azure. Highly recommended for Azure workloads. Provides geo-replication, RBAC, image scanning, etc.
  • Essential for CI/CD pipelines and deploying containers to Azure services.

Why Use Containers?

  • Portability: "Build once, run anywhere" - consistent environments across dev, test, prod, on-prem, or cloud.
  • Efficiency & Speed: Containers share the host kernel, leading to faster startup times and lower resource overhead compared to VMs. More containers can run on a single host.
  • Isolation: Applications run in isolated user spaces, reducing conflicts between dependencies.
  • Consistency: Eliminates "it works on my machine" problems by packaging all dependencies.
  • Scalability: Easy to scale applications up or down by running more or fewer container instances.
  • Microservices: Well-suited for building and deploying microservice architectures.

Running Containers on Azure

Azure provides several services that leverage Docker containers for deploying and managing applications:

  • Azure Container Instances (ACI):
    • Simplest way to run a single Docker container or a small group of co-located containers in Azure without managing underlying VMs.
    • Fast startup, per-second billing. Ideal for simple tasks, batch jobs, or quick tests.
  • Azure Kubernetes Service (AKS):
    • A managed Kubernetes orchestration service. Kubernetes uses Docker (or other OCI-compliant runtimes) to run containerized applications at scale.
    • Manages application lifecycle, scaling, service discovery, load balancing, and more for complex applications.
  • Azure App Service for Containers (Web App for Containers):
    • PaaS offering to deploy web applications packaged as Docker containers.
    • Leverages App Service features like deployment slots, custom domains, auto-scaling, and integrated CI/CD.
  • Azure Container Registry (ACR):
    • A private Docker image registry in Azure. Essential for storing and managing your custom Docker images used by ACI, AKS, App Service, and other services.

AZ-104 Focus: Understand that these Azure services run Docker containers, the purpose of ACR, and the basic use cases for ACI vs. AKS vs. App Service for Containers. You are not expected to be a Docker CLI expert.

Core Understanding for the Exam

  • Docker is a platform for running applications in isolated containers.
  • A Dockerfile builds an Image.
  • An Image is a package of your app and its dependencies.
  • A Container is a running instance of an image.
  • Container Registries (like Azure Container Registry - ACR) store and distribute images.
  • Azure services like ACI, AKS, and App Service for Containers use Docker images.
  • Understand the main benefits: portability, efficiency, consistency.
