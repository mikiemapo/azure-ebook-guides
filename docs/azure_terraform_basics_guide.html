Terraform Basics for Azure

Concise guide to Terraform's role in Azure Infrastructure as Code, especially for AZ-104 lab preparation. Click/Tap headers.

Key Points:

  • Terraform = Infrastructure as Code (IaC) tool by HashiCorp.
  • Uses HashiCorp Configuration Language (HCL) to define infrastructure.
  • Declarative language: You define the *desired end-state* of your Azure resources, and Terraform figures out how to achieve it.
  • Provider-based: Uses specific "providers" (like `azurerm` for Azure) to interact with cloud APIs.

Typical Terraform Files:

  • `main.tf`: Defines your actual Azure resources (e.g., `azurerm_storage_account`, `azurerm_resource_group`).
  • `provider.tf`: Configures the cloud provider(s) and authentication (e.g., `provider "azurerm"` block).
  • `variables.tf`: Defines input variables for customization (e.g., resource group name, location).
  • `terraform.tfstate`: A crucial file that Terraform uses to keep track of the deployed infrastructure's state. Do NOT manually edit this file!

Core Workflow:

  • `terraform init`: Initializes a new or existing Terraform project. Downloads providers (e.g., `azurerm`) and sets up backend state.
  • `terraform plan`: Previews the actions Terraform will take to achieve the desired state. Shows what will be created, modified, or destroyed.
  • `terraform apply`: Executes the planned actions to deploy or update infrastructure in Azure. Requires confirmation.
  • `terraform destroy`: Tears down all resources managed by the current Terraform configuration. Use with extreme caution!

Typical Use Case:

  • Provision an Azure Storage Account (or any other Azure resource) as part of repeatable, version-controlled deployments. Ensures consistency across environments.

AZ-104 Relevance:

  • Understand Terraform basics as part of Infrastructure as Code (IaC) and automation for Azure deployments.
  • Not a core exam topic, but highly helpful for hands-on labs and general DevOps awareness in a cloud environment. It demonstrates efficient, scalable cloud management.

Key Takeaways:

  • Terraform is an IaC tool (HCL, Declarative).
  • Core files: `main.tf`, `provider.tf`, `variables.tf`, `terraform.tfstate`.
  • Key commands: `init`, `plan`, `apply`, `destroy`.
  • Useful for labs & general Azure automation, even if not directly tested on the AZ-104 theory.
