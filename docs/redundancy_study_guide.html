Locally-redundant storage

Basic protection within a single data center.

  • Concept: 3 synchronous copies within one physical building (data center).
  • Copies: 3 (in 1 DC)
  • Scope: Single Data Center
  • Durability: ≥ 99.999999999% (11 nines)
  • Protects against: Drive/Server rack failures.
  • Vulnerable to: Data center-level disaster.
  • Read Access: Primary only.

Zone-redundant storage

Protection across multiple data centers within a region.

  • Concept: 3 synchronous copies spread across 3 different Availability Zones (AZs) in the primary region.
  • Copies: 3 (1 per AZ)
  • Scope: Multiple AZs (1 Region)
  • Durability: ≥ 99.9999999999% (12 nines)
  • Protects against: Data center/AZ failure.
  • Vulnerable to: Region-wide disaster.
  • Read Access: Primary only (any AZ).

Geo-redundant storage

Region-level protection by copying data to a secondary region.

  • Concept: LRS in primary (3 sync copies) + Asynchronous LRS copy in a paired secondary region.
  • Copies: 6 Total (3 Pri + 3 Sec)
  • Scope: Two Regions
  • Durability: ≥ 99.99999999999999% (16 nines)
  • Protects against: Regional disaster.
  • Failover: Manual/Microsoft failover required to access secondary data.
  • Read Access: Primary only (unless failed over).

Read-access geo-redundant storage

GRS plus read-only access to the secondary region.

  • Concept: Same as GRS, but allows read-only access to the secondary region's LRS copy.
  • Copies: 6 Total
  • Scope: Two Regions
  • Durability: ≥ 99.99999999999999% (16 nines)
  • Benefit: Higher read availability, secondary read endpoint.
  • Failover: Still needed for write access in secondary.
  • Read Access: Primary (RW) + Secondary (RO).

Geo-zone-redundant storage

Combines zone redundancy in the primary with geo-replication.

  • Concept: ZRS in primary (3 sync copies across AZs) + Asynchronous LRS copy in secondary.
  • Copies: 6 Total (3 Pri-ZRS + 3 Sec-LRS)
  • Scope: Two Regions (Primary ZRS + Secondary LRS)
  • Durability: ≥ 99.99999999999999% (16 nines)
  • Benefit: AZ failure protection in primary + Regional disaster protection.
  • Failover: Required to access secondary data.
  • Read Access: Primary only (any AZ), unless failed over.

Read-access geo-zone-redundant storage

Maximum redundancy and availability with reads from both regions.

  • Concept: GZRS + read-only access enabled for the secondary region's LRS copy.
  • Copies: 6 Total
  • Scope: Two Regions (Primary ZRS + Secondary LRS+Read)
  • Durability: ≥ 99.99999999999999% (16 nines)
  • Benefit: Highest availability option. AZ + Regional protection + Secondary Reads.
  • Failover: Still needed for write access in secondary.
  • Read Access: Primary (RW, any AZ) + Secondary (RO).
