Azure Storage Account Overview

A comprehensive guide to understanding Azure Storage accounts, types, and configurations. Essential for AZ-104.

What is an Azure Storage Account?

  • Foundation for all Azure Storage services (Blobs, Files, Queues, Tables, Disks).
  • Unique namespace in Azure for your data.
  • Account types: General-purpose v2 (GPv2) (recommended), Premium (for high performance).
  • Supports redundancy and encryption by default.

Key Concepts for the Exam

  • Redundancy options: LRS, ZRS, GRS, RA-GRS, GZRS.
  • Endpoints: Standard and DNS zone endpoints. Know the difference!
  • Billing factors: Region, redundancy, storage tier, type, usage, and data egress.
  • Scalability targets: Limits on storage accounts per region, performance boundaries.
  • Encryption at rest: Always enabled by default.

Exam Tips

  • Choose GPv2 for most workloads unless low-latency is required.
  • Replication: Understand when to use LRS (cheapest, single region) vs. GRS (geo-redundant, for DR).
  • Networking: Use private endpoints and firewalls for secure access.
  • Cost optimization: Use correct access tiers (hot, cool, archive) and monitor egress.
  • Classic storage accounts retire August 31, 2024.
Remember: All storage resources (blobs, files, etc.) must be created inside a storage account!
