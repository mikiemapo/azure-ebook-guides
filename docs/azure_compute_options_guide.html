The Core of the Operating System

The kernel is the central component of an operating system (OS) that manages the system's resources and acts as a bridge between hardware and software applications.

What Does the Kernel Do?

  • Resource Management: Controls CPU scheduling, memory allocation, and access to I/O devices (disks, network).
  • System Calls: Provides an interface for applications to request services from the OS (e.g., read a file, send network data).
  • Process & Memory Isolation: Ensures processes don't interfere with each other or the kernel itself.

Containers share the Host OS Kernel.
  • Virtual Machines (VMs):
    • Each VM runs a full, independent operating system.
    • Therefore, each VM has its own dedicated kernel.
    • This provides strong isolation but means higher resource overhead (CPU, memory, disk space for the full OS).
  • Containers (especially Linux Containers):
    • Containers are isolated user-space processes.
    • They share the kernel of the host operating system they run on (e.g., the kernel of the VM acting as an AKS node or an ACI host).
    • This shared kernel model is why containers are much more lightweight and start faster than VMs â€“ no need to boot a separate OS.
    • (Windows containers have different isolation modes, including Hyper-V isolation which provides a utility VM with its own kernel for stronger isolation, but process isolation shares the host kernel).

Implications for Azure Compute:

  • VMs offer stronger OS-level isolation due to separate kernels.
  • Containers offer greater density and faster startup due to sharing the host kernel.

AZ-104 Focus: Understand this fundamental difference as it explains the resource efficiency and speed benefits of containers compared to VMs. You don't need deep kernel programming knowledge, but the conceptual difference is important for choosing compute options.
