Container Orchestration

An open-source platform for automating deployment, scaling, and management of containerized applications.

  • Containers First: Assumes applications are packaged into lightweight, portable containers (like Docker).
  • Orchestration: Manages the lifecycle of containers across a cluster of machines (nodes).
    • Scheduling containers onto nodes.
    • Handling failures (rescheduling).
    • Scaling applications up or down.
    • Managing network routing and service discovery.
    • Orchestrating storage.
  • Declarative Configuration: You define the desired state (e.g., "run 3 replicas of my app") and Kubernetes works to achieve it.

Core Benefits

Advantages provided by the Kubernetes platform.

  • Scalability: Easily scale applications horizontally (more instances) or vertically (more resources per instance).
  • Self-healing: Automatically restarts failed containers, replaces unresponsive nodes, and reschedules workloads.
  • Portability: Runs consistently across different environments (on-premises, various clouds).
  • Automated Rollouts & Rollbacks: Manages application updates and can automatically roll back if issues occur.
  • Storage Orchestration: Mount and manage various storage systems.
  • Configuration Management: Manage application configuration and secrets separately from container images.

Managed Kubernetes on Azure

AKS simplifies deploying and managing Kubernetes clusters in Azure by handling operational overhead.

Azure Platform
AKS (Managed K8s Control Plane)
Your Containerized Applications
  • Managed Control Plane: Azure manages the complex Kubernetes control plane components (API server, etcd, scheduler, etc.) at no cost.
  • Agent Nodes: You manage and pay for the virtual machines (agent nodes) where your application containers run.
  • Simplification: Handles tasks like health monitoring, patching, scaling (control plane), and upgrades for the control plane.
  • Azure Integration: Seamlessly integrates with other Azure services (Azure AD, Monitor, Policy, Networking, Storage, etc.).

Why Choose Azure's Managed Offering

Advantages of using AKS over managing Kubernetes yourself.

  • Free Control Plane: Reduces operational cost and complexity.
  • Simplified Operations: Azure handles patching, upgrades, scaling, and monitoring of the control plane.
  • Enhanced Security: Integration with Azure AD for RBAC, Azure Policy for governance, Azure Security Center.
  • Flexible Networking: Integrates with Azure Virtual Networks (VNet) using Azure CNI or Kubenet.
  • Monitoring: Easily integrate with Azure Monitor for containers for logging and metrics.
  • Node Pools: Support for multiple node pools (Windows/Linux, different VM sizes, GPU nodes).

  • Node: A worker machine (VM or physical) where containers run. Managed by the control plane.
  • Pod: The smallest deployable unit. Represents a single instance of an application. Can contain one or more tightly coupled containers that share resources (network, storage).
  • Service: Provides a stable IP address and DNS name for a set of pods. Enables network connectivity and load balancing.
  • Deployment: Declaratively manages replica sets of pods. Handles scaling, updates, and rollbacks.
  • Namespace: A way to logically divide cluster resources between multiple users, teams, or applications.

  • Control Plane: The core Kubernetes services making global decisions about the cluster (scheduling, responding to events). Managed by Azure in AKS.
  • Node Pools: Groups of nodes (VMs) with the same configuration. You create and manage these (VM size, OS, count). Backed by Virtual Machine Scale Sets (VMSS).
  • Kubelet: Agent running on each node that ensures containers described in PodSpecs are running and healthy. Communicates with the control plane.
  • Kube-proxy: Network proxy running on each node, maintaining network rules and enabling communication for Services.

  1. User (or CI/CD) defines application state (e.g., Deployment YAML).
  2. Sends request to the API Server (control plane entry point).
  3. Control plane components process the request (e.g., Scheduler finds a suitable node).
  4. API Server updates node's Kubelet.
  5. Kubelet pulls container image(s) and starts the Pod.
  6. Kube-proxy updates network rules for discoverability (if a Service is defined).

Where AKS Shines

Typical scenarios benefiting from Kubernetes and AKS.

  • Microservices Architectures: Easily deploy, scale, and manage independent services.
  • Lift and Shift to Containers: Modernize existing applications by containerizing them.
  • CI/CD Pipelines: Automate building, testing, and deploying applications reliably.
  • Batch Processing: Run short-lived jobs and scale nodes based on workload.
  • Machine Learning: Deploy and scale ML models for training or inference.
  • Web Applications: Run scalable and resilient web frontends and APIs.
