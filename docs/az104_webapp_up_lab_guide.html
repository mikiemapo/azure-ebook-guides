Lab: Deploy Static HTML App with `az webapp up`

This guide reviews your lab session using Azure CLI's az webapp up command to quickly deploy a static HTML website to Azure App Service.

Sequence of CLI Operations

A summary of the steps you performed in the Cloud Shell.

CLI (Cloud Shell): Set defaults (`az configure --defaults location=... group=...`)
CLI: Set variables (`appName=az204app$RANDOM`)
CLI: Attempted `cd html-docs-hello-world` (failed, if current directory was already `~/htmlapp/html-docs-hello-world` or it didn't exist at `~/htmlapp/html-docs-hello-world/html-docs-hello-world`)
CLI: `az webapp up -g $resourceGroup -n $appName --html` (Run from directory containing content to deploy, e.g. `~/htmlapp/html-docs-hello-world` or its parent `~/htmlapp` if that's the source)
Azure automatically:
  • Created App Service Plan (e.g., `mike.mapfumo_asp_6079`).
  • Created Web App (e.g., `az204app26656`).
  • Zipped contents of specified/current directory.
  • Deployed zip to Web App.
Output: Received URL (e.g., `http://az204app26656.azurewebsites.net`) and other resource details.

Simplified Deployment

A powerful convenience command for quickly creating and deploying web apps to Azure App Service.

az webapp up -g <resource_group> -n <app_name> --html (or --sku, --plan, --location)
  • All-in-One: If resources don't exist, `az webapp up` can create them (Resource Group, App Service Plan, Web App).
  • Content Deployment: For --html, it zips the contents of the current directory by default (or --src-path if specified) and deploys it.
  • Defaults: Uses defaults for SKU (often F1 - Free), location, and plan name if not specified. Saves these as local defaults for the directory.

AZ-104 Focus: `az webapp up` for quick deployment, its resource creation ability, how it deploys from local directory.

Web App & App Service Plan

Key components created or used by `az webapp up`.

App Service Plan (Region, Size/SKU, OS)
Web App (Your App) (`name.azurewebsites.net`)
  • Web App: PaaS offering that hosts your web application code/content. Gets a unique URL.
  • App Service Plan: Defines the underlying compute resources (region, VM size/SKU, instance count, OS) for your Web App(s). One plan can host multiple apps. You scale the Plan.

Implication: The Web App runs *on* an App Service Plan. You pay for the Plan (unless Free tier).

Understanding the Results

The JSON output from `az webapp up` provides key details.

{
  "URL": "http://az204app26656.azurewebsites.net",
  "appserviceplan": "mike.mapfumo_asp_6079",
  "location": "canadacentral",
  "name": "az204app26656",
  "os": "Windows", // Default for --html
  "resourcegroup": "learn-8f560196...",
  "sku": "FREE",
  "src_path": "//home//mike_mapfumo//htmlapp//html-docs-hello-world"
}
  • URL: Public web address to access your site.
  • appserviceplan: Name of the plan.
  • location: Azure region.
  • sku: Pricing tier of the App Service Plan (e.g., FREE).
  • src_path: The local directory AzCopy zipped. The `cd` error in your log means `az webapp up` likely ran from `/home/mike_mapfumo/htmlapp/`. If `html-docs-hello-world` was a direct subfolder there, its content was deployed.

Verification: Open the URL in a browser.

AZ-104 Takeaways

  • az webapp up: Quick deployment for simple web apps.
  • Automates App Service Plan and Web App creation.
  • Current working directory matters for what `az webapp up` deploys by default.
  • App Service Plan provides compute; Web App hosts the site.
