Cloud Shell vs. VM Shell

Understanding the difference was the core of solving the lab problem.

CLOUD SHELL
(Azure Environment)
VM SHELL (SSH)
(Inside the VM OS)
  • AZURE CLOUD SHELL:
    • Is an environment hosted by Azure that has Azure CLI (`az`) and Azure PowerShell (`Az`) tools pre-installed.
    • Commands run here (like `az vm ...`) talk to the Azure Resource Manager (ARM) API to manage your Azure resources (like VMs, NSGs, Storage, etc).
  • VM SHELL (via SSH):
    • You are running commands inside the Guest Operating System (Ubuntu, in your case) on that specific VM.
    • It has OS commands (like `sudo`, `apt-get`, `curl`, `ls`).
    • It does NOT have the Azure CLI (`az`) tools installed by default.

To manage an Azure Resource (like modifying the NSG Rules), you MUST run the `az` command from an environment that has the Azure CLI installed and logged in, like the Cloud Shell.

Commands You Used

Controlling the state and configuration of the VM resource via the `az vm` command group.

  • az vm create: Deploys the VM, creating associated NIC, Disk, Public IP, and NSG resources by default if not specified.
  • az vm list / show: Query for deployed VMs and their properties.
  • az vm stop / start / restart: Manage the power state of the VM. (Stop deallocates by default, saving compute costs).
  • az vm resize: Changes the VM Size (CPU/Memory). Requires the VM to be running or stopped, but may require a restart.
  • az vm list-ip-addresses: Convenient way to see Public and Private IPs.

How `az vm open-port` Works

This is a convenience command that modifies the VM's Network Security Group (NSG).

Flow: Before vs After

BEFORE (Port 80 Blocked)
Web Request
NSGDefault Deny Blocks Port 80
VM + Nginx
AFTER (Run `az vm open-port` in Cloud Shell)
Web Request
NSGRule Added to Allow Port 80
VM + Nginx
  • When an Azure VM is created, it usually gets an NSG associated with its Network Interface (NIC).
  • By default, NSGs block most inbound traffic from the Internet for security.
  • The az vm open-port --port 80 ... command, run from Cloud Shell, finds the NSG attached to the VM and adds a new inbound security rule to ALLOW traffic on the specified port (80) from Any source (`*`).
  • Once the NSG rule allows traffic on Port 80, and Nginx is running and listening on Port 80 inside the VM, connections (like from `curl` or a browser) can succeed.

Inside the VM

Once connected, you manage the VM's Guest OS directly.

  • SSH: Used your SSH client (available in Cloud Shell) to connect securely to the Linux VM's OS using its Public IP address.
  • `sudo apt-get update`: Updates the list of available software packages from the configured repositories.
  • `sudo apt-get install nginx`: Installs the Nginx web server software packages onto the VM's operating system.
  • `exit`: Command used to close the SSH session and return to the Cloud Shell prompt.

Tasks performed inside the VM via SSH manage the VM's Guest OS and Software, NOT the Azure resource properties.

Reinforced Concepts

  • Azure CLI (in Cloud Shell) is used to manage Azure Resources (VMs, NSGs, IPs).
  • SSH is used to log in and manage the Guest Operating System inside a VM.
  • The `az` command does not exist inside a standard VM unless you explicitly install it.
  • To allow network traffic to a VM, you must configure its associated Network Security Group (NSG).
  • az vm open-port is a convenient CLI command to modify NSG rules to allow specific inbound ports.
