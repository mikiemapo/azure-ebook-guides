High-Performance CLI Tool

A command-line utility optimized for copying data to/from Azure Blob Storage, Azure Files, and ADLS Gen2.

  • Performance: Designed for speed and reliability, using parallelism for large transfers.
  • Scriptable: Easily integrate into scripts (PowerShell, Bash) for automation.
  • Cross-Platform: Available for Windows, macOS, and Linux.
  • Supports Major Services: Primarily Blob Storage (including ADLS Gen2) and Azure Files. Limited support for Table Storage copy.

Key Advantages

When to choose AzCopy over other methods like the portal or Storage Explorer.

  • Speed for Large Data: Significantly faster for transferring large files or large numbers of files due to parallelism.
  • Automation & Scripting: Ideal for automating regular data transfers or migration tasks.
  • Resume Capability: Can automatically resume failed or interrupted jobs (copy/sync).
  • Synchronization: Efficiently sync local directories with Azure Storage or between Azure locations (only copies changes).
  • Between Accounts/Regions: Efficiently copy data directly between storage accounts without downloading locally.

  • Azure Active Directory (Azure AD) - Recommended:
    • Use `azcopy login` to authenticate interactively (user) or via Service Principal / Managed Identity (scripts).
    • Relies on Azure RBAC roles assigned to the identity (e.g., `Storage Blob Data Contributor`) on the storage account.
    • Most secure method, avoids handling keys or SAS tokens directly in commands.
  • Shared Access Signature (SAS) Token:
    • Append a valid SAS token (providing specific permissions for a limited time) directly to the source/destination URL in the command.
    • Useful for scenarios where Azure AD login isn't feasible or for granting temporary, specific permissions.
    • Requires careful management of the SAS token's expiry and permissions.
  • Storage Account Key (Not Recommended): While technically possible via environment variables, using account keys grants full access and is generally discouraged for security reasons. Prefer Azure AD or SAS.

Transferring Files/Blobs

The primary command for uploading, downloading, and copying between Azure locations.

azcopy copy '<source>' '<destination>' [flags]
  • Upload: `azcopy copy '/path/to/local/folder' 'https://[account].blob.core.windows.net/[container]/[path]' --recursive`
  • Download: `azcopy copy 'https://[account].blob.core.windows.net/[container]/[blob]' '/path/to/local/file'`
  • Account-to-Account (Blob): `azcopy copy 'https://[source].blob.core.windows.net/[container]' 'https://[dest].blob.core.windows.net/[container]' --recursive`
  • Common Flags:
    • --recursive: Copy directory contents.
    • --include-pattern / --exclude-pattern: Filter files.
    • --blob-type: Specify BlockBlob, PageBlob, AppendBlob.
    • --block-blob-tier: Set tier on upload/copy.

Copies all specified source files/blobs, potentially overwriting the destination.

More Capabilities

  • azcopy login: Authenticate using Azure AD.
  • azcopy logout: Clear cached Azure AD tokens.
  • azcopy make: Create a container or file share. (`azcopy mk`)
  • azcopy list: List contents of a container or share.
  • azcopy remove: Delete blobs or files. (`azcopy rm`)
  • Jobs Management: View (`azcopy jobs list`), resume (`azcopy jobs resume`), or cancel (`azcopy jobs clean`) transfer jobs.
  • Automatic Resume: AzCopy automatically attempts to resume failed/interrupted transfers when restarted.
  • Logging & Plan Files: Generates detailed logs and plan files (for resuming) for troubleshooting. Configure log level with flags.
  • Performance Tuning: Adjust concurrency (`AZCOPY_CONCURRENCY_VALUE`) and other environment variables.
